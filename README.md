# AI Project Governance Framework
### For Business Analysts Leading AI Delivery

## Why Governance Matters More Than the Model
Most AI projects fail not because of bad models — but because of:
- Unclear success criteria (what does "good" look like?)
- Missing fallback processes (what happens when AI is wrong?)
- No monitoring plan (how do you detect drift post-deployment?)
- Undefined accountability (who owns what when it fails?)

This framework addresses all four.

## Framework Sections
1. **RACI Matrix** — Role accountability for every governance activity
2. **Data Readiness Checklist** — Green/Amber/Red status per data requirement
3. **Model Acceptance Criteria** — Accuracy thresholds, latency, explainability
4. **Ethics Checklist** — Bias, explainability, right to human review
5. **Go-Live Scorecard** — Pass/Review/Fail gates before production
6. **Post-Deployment Monitoring** — Metrics, frequency, alert thresholds

## Origin
Built from real project experience:
- ARTP (Citi): Silent bot failure due to missing error handling → added alert requirements
- Amazon EU: No drift detection post-deployment → added monitoring plan
- Santander AML: Inconsistent rule application → added model acceptance criteria

## Use This Framework When
- Starting any AI or automation project
- Preparing for stakeholder governance review
- Defining UAT acceptance criteria for AI outputs
- Planning post-deployment monitoring
